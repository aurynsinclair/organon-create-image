## Step 1: セッションログの記録

今日の日付から `D:\opus_meum\Nexus\log\YYYY-MM-DD-viola-organon-create-image.md` のパスを決定し、Globで既存ファイルの有無を確認してください。

- **既存ファイルがない場合：** 新規作成してください。

## Step 2: Git commit & push

プロジェクトに未コミットの変更がある場合：

1. `git status` で変更を確認
2. 変更内容を見て適切にステージング
3. 簡潔な英語コミットメッセージを作成して `git commit`
4. `git push`

変更がなければスキップ。

## Step 3: Nexus を Git commit & push

`git -C` を使って Nexus リポジトリを操作してください（`cd` ではなく `git -C` を使うこと。`Bash(git:*)` の allowlist パターンにマッチさせるため）：

1. `git -C "D:\opus_meum\Nexus" add -A`
2. 変更内容から簡潔な英語コミットメッセージを作成して `git -C "D:\opus_meum\Nexus" commit -m "..."` — Co-Authored-By 等の追加行は `-m` フラグを複数回使うこと（heredoc や `$(cat ...)` はコマンド置換内の `cat` が allowlist 外となり ask される）
3. `git -C "D:\opus_meum\Nexus" push`
